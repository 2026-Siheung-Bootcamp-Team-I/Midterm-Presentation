# 1. 서비스 설명 — 기획 배경 & 문제 정의

## 왜 만드는가?

### 기존 백신(안티바이러스)의 한계
- 기존 백신은 **알려진 유명 바이러스, 악성 파일만** 잡을 수 있음
- 새로운 공격, 알려지지 않은 이상 행동에는 대응 불가

### EDR의 접근 방식
- EDR(Endpoint Detection & Response)은 **컴퓨터 행동의 특이점**을 발견하여 보안을 실행

> **비유**
> - 백신 = 감기약 보유 여부로 감기를 판단 (알려진 병만 진단)
> - EDR = **증상으로 판단** (처음 보는 병도 이상 징후로 탐지)

## 우리가 만드는 것
- 이러한 장점을 가진 **EDR 방식의 보안 프로그램**을 직접 제작
- **RAM, CPU, 네트워크, 파일, DNS** 등에서의 이상 행동을 탐지하고 사용자에게 안내
- 기존 백신 대비 효과적인 **사전 예방** 중심의 보안 해결책 제시
