# 10. 기술 ③ 알림 — Spring Boot · Slack

> 🎬 **시나리오 ④**: 위협으로 판정된 순간, Spring Boot가 즉시 Slack으로 "수상한 로그인 시도 감지" 알림을 보낸다.
>
> 🎨 **시각화 메모**: 아키텍처 그림에서 `Kafka(alerts) → Alert Service(Spring Boot) → Slack` 부분만 선명하게, 나머지는 흐리게 처리해 현재 위치 강조.

## 동작 원리 3단계

### ① Incoming Webhook — 메시지를 넣는 고유 URL
- Slack 채널과 1:1로 매핑된 **고유 URL**을 발급받아 사용
- 일종의 **'비밀 우체통' 주소** — 규격에 맞는 데이터를 넣으면 Slack이 채널에 자동 표시

### ② JSON 데이터 페이로드 — 데이터 규격화
- Slack은 지정된 **JSON 형식**만 인식 (가장 단순한 형태: `{"text": "보낼 내용"}`)

### ③ HTTP 발송 — Spring Boot
- 위협 탐지 시 Spring Boot가 Webhook URL로 JSON을 담아 **POST 요청**

## 전체 흐름

```
[이벤트 발생] → [Spring Boot에서 JSON 조립] → [WebClient로 Webhook URL에 POST] → [Slack 채널에 알림 표시]
```
