# 7. 공격 시나리오 — "수상한 로그인을 잡아라"

> 앞의 아키텍처를 이 공격 하나로 따라간다. 다음 기술 슬라이드에서 각 단계(①~⑥)를 이어서 설명한다.

## 상황

**한밤중, 누군가 내 컴퓨터에 비밀번호를 100번 넘게 찍어보고 있다.**
(정상 사용자는 이렇게 안 함 → 명백한 이상 행동)

## 5초 안에 잡는 과정

| 단계 | 무슨 일이 일어나는가 | 담당 기술 |
| --- | --- | --- |
| ① 수집 | "로그인 실패" 기록이 계속 쌓임 → 실시간으로 긁어모음 | OSquery / Zeek |
| ② 전달 | 이 기록들을 한 줄로 흘려보냄 | Kafka |
| ③ 탐지 | 길목에서 감시: "5초에 50번 넘게 실패!" → 위협 판정 | Kafka Streams |
| ④ 알림 | 즉시 Slack으로 "수상한 로그인 시도 감지" 전송 | Spring Boot |
| ⑤ 저장 | 모든 기록은 창고에 보관 | ClickHouse |
| ⑥ 시각화 | 대시보드에서 언제 몇 번 시도했는지 확인 | React |

## 한 문장

> 누가 비밀번호를 무차별로 찍어보면, 우리 시스템이 **5초 만에 알아채고 Slack으로 알려준다.**
