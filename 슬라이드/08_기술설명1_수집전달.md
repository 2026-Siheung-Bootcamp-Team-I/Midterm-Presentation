# 8. 기술 ① 수집·전달 — OSquery / Zeek / Kafka

> 🎬 **시나리오 ①②**: 누가 비밀번호를 100번 찍어보면 "로그인 실패" 기록이 계속 쌓인다. OSquery/Zeek가 이 기록을 실시간으로 긁어모아 Kafka로 흘려보낸다.
>
> 🎨 **시각화 메모**: 아키텍처 그림에서 `OSquery/Zeek → Kafka(events)` 부분만 선명하게, 나머지는 흐리게 처리해 현재 위치 강조.

## OSquery
- Meta(메타)에서 만든 보안·모니터링 프로그램
- PC 내부(RAM, CPU, 프로세스 등)의 행동을 수집하고 **Kafka로 전달**

### 장점
- 복잡한 보안 문법 필요 없음 — 일반적인 데이터베이스 언어 **SQL 문법** 사용
- **멀티플랫폼 지원** — Windows, Mac 등 여러 운영체제에서 폭넓게 사용 가능
- 실시간으로 **RAM, CPU, 네트워크** 등의 정보 탐색

## Zeek
- 네트워크 트래픽 전용 모니터링 도구
- OSquery가 PC 내부를 본다면, Zeek은 **네트워크 행동**을 실시간 기록

## Kafka
- 수집된 로그가 모이는 **실시간 메시지 큐** (`events` 토픽)
- 하나의 로그를 **Kafka Streams(탐지)와 ClickHouse(저장)에 동시 전달** 가능
- 짧은 시간에 쏟아지는 대량의 로그를 유실 없이 버퍼링
