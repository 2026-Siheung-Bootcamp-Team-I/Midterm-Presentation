# 9. 기술 ② 탐지 — Kafka Streams

> 🎬 **시나리오 ③**: 흘러오는 로그인 실패 기록을 길목에서 감시하다 "5초에 50번 넘게 실패!"를 포착 → 위협으로 판정하고 `alerts` 토픽으로 넘긴다.
>
> 🎨 **시각화 메모**: 아키텍처 그림에서 `Kafka Streams(group: detector)`만 선명하게, 나머지는 흐리게 처리해 현재 위치 강조.

## 역할
- Kafka로 들어오는 데이터를 DB에 저장하기 전, **흘러가는 길목(메모리)에서 실시간으로 검사·가공**하는 도구

## 기능
- **실시간 필터링**: 쏟아지는 원시 로그에서 필요한 조건(실패 로그, 위협 패턴)만 즉시 골라냄
- **윈도우 연산**: *"최근 5초 동안 50번 이상 실패했는가?"* 처럼 시간 범위 내 데이터를 실시간 집계
- **데이터 변환**: 가공이 끝난 이상 데이터만 다음 토픽(`alerts`)으로 전달

## 사용해야 하는 이유
- **초고속 반응**: DB 저장 후 조회하는 일반 방식과 달리, 사건 발생 즉시 **0.1초 내외로 Slack 알림** 가능
- **서버 부하 감소**: 이상 감지를 메모리에서 처리 → DB 과부하 방지

> **보통 방식 vs Kafka Streams**
> - 보통 방식: 일단 DB에 저장 → 주기적으로 조회 → 시간이 걸려 실시간 대응 불가
> - Kafka Streams: 데이터가 지나가는 순간 바로 검사 → **DB에 들어가기 전에 즉시 알림**
