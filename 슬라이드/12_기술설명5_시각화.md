# 12. 기술 ⑤ 시각화 — React 대시보드

> 🎬 **시나리오 ⑥**: 창고에 쌓인 기록을 대시보드로 보여준다. "언제 몇 번 로그인을 시도했는지" 그래프와 목록으로 확인한다.
>
> 🎨 **시각화 메모**: 아키텍처 그림에서 `ClickHouse → API Server(Spring Boot) → React` 부분만 선명하게, 나머지는 흐리게 처리해 현재 위치 강조.

## 구조

- **ClickHouse → Spring Boot → React** 순서로 데이터 전달
- ClickHouse의 데이터를 Spring Boot(API 서버)가 **React가 요청하는 자료 규격(API)** 으로 반환
- React는 Spring Boot에서 받은 API를 토대로 화면 출력

## 이런 화면으로 보여집니다 (앞의 UI 레퍼런스 참고)

- **메인 대시보드**: 보안 점수 / 이벤트 추이 그래프 / 최근 탐지 목록
- **이벤트 상세**: 탐지 시각·위치·위협 유형 + 요약 팝업
- **로그 조회**: ClickHouse에 저장된 원본 로그 검색·필터링

> 이번 발표에서는 직접 시연 대신, **5번에서 본 UI 구성으로 이렇게 만들 계획**임을 보여준다.
> ClickHouse의 빠른 집계 덕분에 대용량 로그도 실시간에 가깝게 시각화된다.
